# 售后表单部署文档

> 适用模块：C 端报修 / 售后表单（Repair）
> 文档覆盖：上传存储、工单回调、工单验收、部署验收命令。

## 一、概述

报修（售后）表单前端位于 [Repair.vue](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/009-C端续租退租报修发票/frontend/src/views/Repair.vue)，后端控制器位于 [repairController.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/009-C端续租退租报修发票/backend/controllers/repairController.js)，路由定义位于 [repair.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/009-C端续租退租报修发票/backend/routes/repair.js)。

本次部署补充以下内容：

1. **上传存储**：使用 multer 落盘存储 + 静态资源服务，替换原 base64 模拟上传。
2. **工单回调**：`POST /api/repair/callback`，供派单 / 维修系统回推工单状态。
3. **工单验收**：`POST /api/repair/accept`，租户对维修结果进行验收（通过 / 驳回）。
4. **部署验收命令**：`npm run verify` 验收脚本 + curl 验收命令。

## 二、环境要求

- Node.js >= 18（验收脚本使用全局 `fetch` / `FormData` / `Blob`）
- npm
- 端口：后端 `3000`，前端 `5173`（开发模式）

### 5.2 接口

`POST /api/repair/upload-image`（`multipart/form-data`，字段 `file`）

请求示例（curl）：

```bash
curl -X POST http://localhost:3000/api/repair/upload-image \
  -F "file=@/path/to/image.png"
```

响应示例：

```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "url": "/uploads/repair/1700000000000-123456789.png",
    "name": "image.png",
    "type": "image",
    "size": 12345
  }
}
```

### 5.3 生产部署注意

- 上传目录需持久化、具备写入权限，建议挂载独立数据卷。
- 若前端与后端不同源，需在 Nginx 将 `/uploads` 反向代理到后端，或由后端统一提供静态资源。
- 生产环境建议对上传目录做定期清理与容量监控。

## 六、工单回调

`POST /api/repair/callback`

用于派单系统 / 维修端回推工单状态：更新工单状态并写入状态轨迹。控制器见 [repairController.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/009-C端续租退租报修发票/backend/controllers/repairController.js) 的 `workOrderCallback`。

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| orderId | string | 是 | 工单 ID |
| status | string | 否 | `pending` / `in_progress` / `completed` / `cancelled` |
| operator | string | 否 | 操作人，默认「系统回调」 |
| description | string | 否 | 轨迹描述 |
| workerName | string | 否 | 维修人姓名 |
| workerPhone | string | 否 | 维修人电话 |
| fee | number | 否 | 维修费用 |

请求示例：

```bash
curl -X POST http://localhost:3000/api/repair/callback \
  -H "Content-Type: application/json" \
  -d '{
    "orderId": "BX20260115001",
    "status": "completed",
    "operator": "李师傅",
    "description": "维修完成，制冷正常",
    "workerName": "李师傅",
    "workerPhone": "139****1234",
    "fee": 180
  }'
```

响应示例：

```json
{
  "code": 200,
  "message": "工单回调处理成功",
  "data": {
    "orderId": "BX20260115001",
    "status": "completed",
    "statusText": "已完成",
    "statusTrack": {
      "status": "completed",
      "statusText": "已完成",
      "time": "2026/6/20 12:00:00",
      "description": "维修完成，制冷正常",
      "operator": "李师傅"
    }
  }
}
```

## 七、工单验收

`POST /api/repair/accept`

租户对维修结果进行验收。控制器见 [repairController.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/009-C端续租退租报修发票/backend/controllers/repairController.js) 的 `acceptRepair`。

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| orderId | string | 是 | 工单 ID |
| result | string | 是 | `accepted`（验收通过）/ `rejected`（验收驳回） |
| comment | string | 否 | 验收说明 |

规则：

- `accepted`：工单置为「已完成」，写入「验收通过」轨迹。
- `rejected`：工单置为「维修中」（返工），写入「验收未通过」轨迹。

请求示例：

```bash
curl -X POST http://localhost:3000/api/repair/accept \
  -H "Content-Type: application/json" \
  -d '{
    "orderId": "BX20260220002",
    "result": "accepted",
    "comment": "维修正常，验收通过"
  }'
```

响应示例：

```json
{
  "code": 200,
  "message": "验收通过，工单已完成",
  "data": {
    "orderId": "BX20260220002",
    "result": "accepted",
    "status": "completed",
    "statusText": "已完成",
    "statusTrack": {
      "status": "accepted",
      "statusText": "验收通过",
      "time": "2026/6/20 12:00:00",
      "description": "租户验收通过：维修正常，验收通过",
      "operator": "张三"
    }
  }
}
```

## 八、部署验收命令

### 8.1 脚本验收

```bash
cd backend
npm run verify
```

- 默认验收目标：`http://localhost:3000/api`
- 可用环境变量覆盖：`API_BASE`（接口地址）、`UPLOAD_ORIGIN`（上传文件访问地址）

示例：

```bash
API_BASE=http://127.0.0.1:3000/api UPLOAD_ORIGIN=http://127.0.0.1:3000 npm run verify
```

验收项：

1. 健康检查 `/api/health`
2. 报修信息 `/repair/info`
3. 工单列表 `/repair/orders`
4. 工单详情 `/repair/order-detail`
5. 上传存储 `/repair/upload-image`（含上传文件可访问性校验）
6. 工单回调 `/repair/callback`
7. 工单验收 `/repair/accept`

> 说明：验收脚本会写入状态轨迹（针对演示工单），属正常现象；后端重启后内存数据恢复初始状态。

### 8.2 curl 验收命令

健康检查：

```bash
curl http://localhost:3000/api/health
```

上传存储：

```bash
curl -X POST http://localhost:3000/api/repair/upload-image -F "file=@./test.png"
```

工单回调：

```bash
curl -X POST http://localhost:3000/api/repair/callback \
  -H "Content-Type: application/json" \
  -d '{"orderId":"BX20260115001","status":"completed","operator":"验收","description":"部署验收-回调测试"}'
```

工单验收：

```bash
curl -X POST http://localhost:3000/api/repair/accept \
  -H "Content-Type: application/json" \
  -d '{"orderId":"BX20260220002","result":"accepted","comment":"部署验收-验收通过"}'
```
